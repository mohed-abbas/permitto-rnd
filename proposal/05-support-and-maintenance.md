# Section 5: Support & Maintenance

---

## 5.1 Support Phases

| Phase | Period | Description |
|-------|--------|-------------|
| **Hypercare** | First 4 weeks after go-live | Intensive post-launch support — priority response to any issues, daily monitoring, immediate bug fixes |
| **Active Support** | Ongoing (after hypercare) | Continued maintenance, bug fixes, security updates, server monitoring, and technical assistance |

---

## 5.2 Hypercare Period (Weeks 1-4 After Go-Live)

Dedicated intensive support immediately after production deployment to ensure smooth adoption.

| Aspect | Detail |
|--------|--------|
| Duration | 4 weeks from go-live date |
| Availability | Sunday–Thursday, business hours (or as agreed) |
| Response Time | Critical issues: within 2 hours. Other issues: within 4 hours |
| Monitoring | Daily system health checks — server, database, background jobs, email delivery |
| Scope | Bug fixes, performance issues, workflow adjustments, user support, configuration changes |
| Communication | Direct channel (WhatsApp/Teams/email) with the development team |

---

## 5.3 Active Support & Maintenance (Ongoing)

Continued support after the hypercare period to keep the system running smoothly, secure, and up to date.

### What's Included

| Category | Coverage |
|----------|---------|
| **Bug Fixes** | Identification and resolution of software defects — no additional charge for bugs in delivered features |
| **Security Updates** | Regular security patches for dependencies, frameworks, and server software |
| **Server Monitoring** | Monitoring of application health, database performance, background job execution, and disk usage |
| **Database Maintenance** | Routine database optimization, backup verification, and performance tuning |
| **Uptime Monitoring** | System availability monitoring with alerts for downtime |
| **Backup Management** | Automated daily database backups with verification |
| **SSL Certificate Management** | SSL/TLS certificate renewal and configuration |
| **Email Service Monitoring** | Monitoring email delivery success rates and troubleshooting delivery issues |
| **Background Job Monitoring** | Ensuring scheduled jobs (daily reminders, suspension checks, expiry warnings) execute reliably |
| **Infrastructure Management** | VPS server, domain, and email service (Resend) — provisioned, managed, and renewed by us. Recurring costs included in the monthly maintenance fee |
| **Minor Adjustments** | Small configuration changes, wording updates, notification timing adjustments |

### Service Level Agreement (SLA)

| Priority | Description | Response Time | Resolution Target |
|----------|-------------|---------------|-------------------|
| **Critical** | System down, no users can access, data loss risk | Within 2 hours | Within 8 hours |
| **High** | Major feature broken (approvals not working, PDFs not generating, notifications failing) | Within 4 hours | Within 24 hours |
| **Medium** | Feature partially impaired, workaround available | Within 8 hours | Within 3 business days |
| **Low** | Minor UI issue, cosmetic defect, non-urgent request | Within 24 hours | Within 5 business days |

### Support Availability

| Aspect | Detail |
|--------|--------|
| Support Hours | Sunday–Thursday, 9:00 AM – 6:00 PM (client's timezone) |
| Emergency Support | Critical issues (system down) — available outside business hours via emergency contact |
| Communication Channels | Dedicated support channel (WhatsApp/Teams/email — as agreed with client) |
| Issue Reporting | Client reports issues via agreed channel with description and screenshots |

---

## 5.4 System Updates & Improvements

| Type | Included in Maintenance | Description |
|------|------------------------|-------------|
| Bug Fixes | Yes | Defects in delivered features |
| Security Patches | Yes | Dependency and server updates |
| Minor Adjustments | Yes | Configuration, wording, timing changes |
| Performance Optimization | Yes | Query optimization, caching improvements |
| New Features / Enhancements | No — quoted separately | New modules, workflow changes, additional permit types, integrations |
| Major Scope Changes | No — quoted separately | Significant functionality additions beyond original scope |

New features and enhancements beyond the delivered scope can be requested and will be estimated and quoted separately.

---

## 5.5 Backup & Disaster Recovery

| Aspect | Detail |
|--------|--------|
| Database Backups | Automated daily backups with 30-day retention |
| Backup Storage | Stored separately from production server for redundancy |
| Recovery Time Objective (RTO) | System restored within 4 hours of a critical failure |
| Recovery Point Objective (RPO) | Maximum 24 hours of data loss (from last daily backup) |
| Disaster Recovery Test | Backup restoration tested quarterly to verify integrity |
